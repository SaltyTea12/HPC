# Лабораторная работа "Salt and Pepper"

### Медианная фильтрация изображений с шумом "Соль и перец"

**Задача:** реализовать медианный фильтр 9 точек (окно 3×3) для удаления шума "Соль и перец" с использованием CPU и GPU (CUDA).  
**Язык:** C++  
**Входные данные:** BMP изображение (цветное или grayscale)  
**Выходные данные:** зашумленное изображение + отфильтрованные изображения (CPU и GPU) + время выполнения + метрики качества (RMSE) + ускорение.  
**Особенность:** CUDA реализация должна использовать texture memory.

## Описание работы

Реализован алгоритм медианной фильтрации изображения на языке C++ с использованием CPU и GPU (CUDA). Для работы с BMP изображениями использована библиотека EasyBMP.

## Платформа и оборудование

- **Графический процессор (GPU):** NVIDIA GeForce 940MX (Compute Capability 5.0)
- **CUDA версия:** 11.8
- **Процессор (CPU):** Intel(R) Core(TM) i3-6006U @ 2.00GHz
- **Оперативная память:** 8 ГБ
- **Среда разработки:** Visual Studio 2022

## Реализация

### Медианный фильтр на CPU

Функция `medianFilterCPU` реализует классическую последовательную версию фильтра. Для каждого пикселя (x, y):
- Извлекается окно 3×3 (9 пикселей) с обработкой границ (дублирование крайних пикселей)
- Значения собираются в массив и сортируются с помощью `std::sort`
- Медиана (5-й элемент отсортированного массива) записывается в выходное изображение

Алгоритм имеет сложность O(H × W × 9 log 9).

### Медианный фильтр на GPU с texture memory

Функция `medianFilterGPU` представляет собой CUDA-ядро. Особенности реализации:

- **Texture memory:** входные данные привязаны к текстуре (`texInput`), доступ через `tex2D`
- **Распараллеливание:** сетка блоков 16×16 нитей, каждый поток обрабатывает один пиксель
- **Размер окна:** 3×3 (9 пикселей)
- **Сортировка:** пузырьком (bubble sort) для 9 элементов
- **Обработка границ:** дублирование крайних пикселей через проверку условий

Технические детали:
- Выделение памяти с pitch (`cudaMallocPitch`) для корректного выравнивания данных
- Привязка данных к текстуре через `cudaBindTexture2D`
- Запуск ядра с размерами сетки `gridSize = (ceil(width/16), ceil(height/16))`
- Отвязывание текстуры после выполнения

### Метрика качества

Для оценки качества фильтрации используется RMSE (Root Mean Square Error):

$$RMSE = \sqrt{\frac{1}{N} \sum (I_{\text{original}}[i] - I_{\text{filtered}}[i])^2}$$

Вычисляются следующие метрики:
- RMSE (Original vs Noisy) — уровень шума
- RMSE (Original vs CPU) — качество CPU фильтрации
- RMSE (Original vs GPU) — качество GPU фильтрации
- RMSE (CPU vs GPU) — идентичность результатов

## Анализ результатов

### Время выполнения
- GPU время (1 мс) значительно меньше CPU времени (121 мс)
- Ускорение составляет 121x
- Texture memory обеспечивает эффективный доступ к пикселям без кэширования конфликтов

### Качество фильтрации
- RMSE между оригиналом и зашумленным изображением высокий (47.9971) из-за добавленного шума
- После фильтрации RMSE снижается до 9.1093 — шум успешно удален
- RMSE между CPU и GPU равен 0.0000 — результаты идентичны, реализация корректна

### Причины высокого ускорения
- Операция медианной фильтрации хорошо параллелится (каждый пиксель независим)
- Texture memory оптимизирует доступ к соседним пикселям за счет аппаратного кэширования
- Отсутствие гонок данных и атомарных операций
