# Задание
### Перемножение матриц
**Задача:** реализовать алгоритм сложения элементов вектора  
**Язык:** C++ или Python  
**Входные данные:** Вектор размером 1 000..1 000 000 значений.  
**Выходные данные:** сумма элементов вектора + время вычисления  
Реализация должна содержать 2 функции сложения элементов вектора: на CPU и на
GPU с применением CUDA.

## Описание работы

Реализован алгоритм суммирования элементов вектора на языке C++ с использованием CPU и GPU (CUDA).

## Платформа и оборудование

- **Графический процессор (GPU):** NVIDIA GeForce 940MX (Compute Capability 5.0)
- **CUDA версия:** 11.8
- **Процессор (CPU):** Intel(R) Core(TM) i3-6006U @ 2.00GHz
- **Оперативная память:** 8 ГБ
- **Среда разработки:** Visual Studio 2022

## Реализация

### Сложение на CPU
В функции `vector_sum_cpu` используется последовательный цикл. Каждый элемент вектора добавляется к общей сумме. Алгоритм имеет сложность O(N).

### Сложение на GPU
Функция `vector_sum_kernel` представляет собой CUDA-ядро. Распараллеливание достигается за счет следующих механизмов:
- Сетка блоков и нитей
- Редукция в разделяемой памяти (shared memory)
- Каждый блок суммирует свои элементы, затем частичные суммы объединяются на CPU
- Отсутствие атомарных операций за счет отдельных массивов частичных сумм

### Параметры тестирования
- Размеры вектора: 1000, 5000, 10000, 50000, 100000, 500000, 1000000
- Количество прогонов для усреднения: 10
- Типы данных: int, float, double
- Заполнение: случайные числа в диапазоне [1, 100] для int, [0, 1] для float и double

## Анализ результатов

### Время выполнения
- Время на CPU растет линейно с увеличением размера вектора (от 0.001 мс до 2.29 мс)
- Время на GPU значительно больше для малых размеров из-за накладных расходов на запуск ядра и копирование данных
- Время ядра GPU сравнимо с временем CPU на больших размерах

### Ускорение
- Полное ускорение (с учетом копирования данных) меньше 1 для всех размеров
- Ускорение только ядра достигает ~1.04 для float и ~1.09 для double на максимальном размере
- Для int ускорение ядра ниже из-за меньшей вычислительной интенсивности
- Ускорение растет с увеличением размера вектора

### Причины низкого ускорения
- Операция сложения ограничена пропускной способностью памяти, а не вычислительными возможностями GPU
- Накладные расходы на копирование данных доминируют над временем вычислений
- Простая арифметическая операция не позволяет полностью загрузить вычислительные блоки GPU
