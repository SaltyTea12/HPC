# Задание
### Перемножение матриц
**Задача:** реализовать алгоритм перемножения матриц  
**Язык:** C++ или Python  
**Входные данные:** 2 матрицы размером от 100х100 до 2000х2000 каждая  
**Выходные данные:** проверка корректности перемножения + время вычисления  
Реализация должна содержать 2 функции перемножения матриц: на CPU и на GPU с
применением CUDA.

## Описание работы
Реализован алгоритм перемножения квадратных матриц на языке C++ с использованием CPU и GPU (CUDA).

## Платформа и оборудование
- **Графический процессор (GPU):** NVIDIA GeForce 940MX (Compute Capability 5.0)
- **CUDA версия:** 11.8
- **Процессор (CPU):** Intel(R) Core(TM) i3-6006U @ 2.00GHz
- **Оперативная память:** 8 ГБ
- **Среда разработки:** Visual Studio 2022

## Реализация

### Умножение на CPU
В функции `matmul_cpu` используется тройной вложенный цикл. Каждый элемент результирующей матрицы вычисляется как скалярное произведение соответствующей строки матрицы A и столбца матрицы B. Алгоритм имеет сложность O(N³).

### Умножение на GPU
Функция `matmul_gpu` представляет собой CUDA-ядро. Распараллеливание достигается за счет следующих механизмов:
- Сетка блоков (`dim3 blocks`) и нитей (`dim3 threads`)
- Каждая нить GPU вычисляет один элемент результирующей матрицы
- Индексы `blockIdx` и `threadIdx` определяют положение нити в сетке
- Запуск ядра осуществляется с синтаксисом `<<<blocks, threads>>>`

### Параметры тестирования
- Размеры матриц: 100, 200, 400, 800, 1000, 1600, 2000
- Количество прогонов для усреднения: 3
- Типы данных: int, float, double
- Проверка корректности: сравнение результатов CPU и GPU с точностью 0.001

## Анализ результатов

### Время выполнения
- Время на CPU быстро растет с увеличением размера матрицы (от 0.96 мс до 51976 мс для int)
- Время на GPU растет значительно медленнее (от 0.45 мс до 661 мс для double)
- Для больших матриц (2000x2000) GPU выполняется в 100+ раз быстрее CPU

### Ускорение
- Ускорение растет с увеличением размера матрицы
- Максимальное ускорение достигнуто для float (112.76x)
- Наименьшее ускорение наблюдается для double (93.22x), что связано с особенностями вычислений двойной точности
- Аномалия при размере 400x400 для float и double связана с особенностями кэширования
